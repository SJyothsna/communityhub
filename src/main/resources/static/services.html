📋 Controller Overview

Base Path: /api/services

This controller manages individual service offerings. Each service belongs to a provider and a subcategory, and can be tagged to multiple communities.

GET /api/services
Get all services
Retrieves a list of all services in the system.
GET /api/services/{id}
Get service by ID
Retrieves a specific service by its ID.
GET /api/providers/{providerId}/services
Get all services for a provider
Retrieves all services offered by a specific provider.
POST /api/services
Create a new service
Creates a new service offering with details.

Service Modes: HOME_VISIT, ONLINE, CENTER_BASED, HYBRID

PUT /api/services/{id}
Update an existing service
Updates service information for the specified ID.
DELETE /api/services/{id}
Delete a service
Permanently deletes a service from the system.

⚠️ Note

This controller needs to be implemented in the backend. The endpoints shown here are the standard CRUD operations that should be created.
