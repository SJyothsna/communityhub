🔗 Community Services Controller

Test endpoints for managing service-community mappings

📋 Controller Overview

Base Path: /api

This controller manages the many-to-many relationship between services and communities. It allows you to:

  • Tag services to communities
  • Retrieve all services for a specific community
  • Remove services from communities
  • Find all communities where a service is available
POST /api/community-services
Tag a service to a community
Creates a mapping between a service and a community. This allows the same service to appear in multiple communities.
GET /api/communities/{communityId}/services
Get all services for a community (MAIN API)
Retrieves all services available in a specific community with optional filtering by category, subcategory, or keyword.
DELETE /api/communities/{communityId}/services/{serviceId}
Remove a service from a community
Deletes the mapping between a service and a community. The service itself is not deleted, only the association.
GET /api/services/{serviceId}/communities
Get all communities where a service is available
Retrieves all communities that have access to a specific service.

⚠️ Prerequisites

Before testing these endpoints, ensure you have:

  1. Created at least one community
  2. Created categories and subcategories
  3. Created at least one provider
  4. Created at least one service

You can create these using the H2 console or by implementing additional controllers.
