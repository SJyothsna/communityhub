📋 Controller Overview

Base Path: /api/subcategories

This controller manages subcategories which are specific service types under a parent category. Examples: "Plumber", "Electrician" under "Home Services", or "Music", "Art" under "Kids Classes".

GET /api/subcategories
Get all subcategories
Retrieves a list of all subcategories in the system.
GET /api/subcategories/{id}
Get subcategory by ID
Retrieves a specific subcategory by its ID.
GET /api/categories/{categoryId}/subcategories
Get all subcategories for a category
Retrieves all subcategories that belong to a specific parent category.
POST /api/subcategories
Create a new subcategory
Creates a new subcategory under a parent category.
PUT /api/subcategories/{id}
Update an existing subcategory
Updates subcategory information for the specified ID.
DELETE /api/subcategories/{id}
Delete a subcategory
Permanently deletes a subcategory from the system.

⚠️ Note

This controller needs to be implemented in the backend. The endpoints shown here are the standard CRUD operations that should be created.
